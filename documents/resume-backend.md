### [불마켓랩스] 인증 서비스 구축 및 MSA 마이그레이션
**Situation**  
기존 멀티 모듈 모놀리식 구조에서 공통 모듈에 대한 의존성이 비정상적으로 높아, 인증 로직의 작은 수정이 전사 서비스의 영향도로 이어지는 유지보수의 임계점에 도달해 있었습니다. 특히 세션 기반 인증 방식이 온프레미스 Redis 트래픽의 30%를 점유하며 시스템 전체의 가용성을 위협하고 있었으며, 파편화된 예외 처리 로직으로 인해 배포 생산성이 크게 저하된 상태였습니다.

**Task**  
'로그인 개편' 프로젝트의 스토리 오너로서 4주(2개 스프린트)라는 타이트한 일정 내에 인증 도메인을 물리적으로 격리하고 현대적인 아키텍처로 마이그레이션하는 전 과정을 리딩했습니다. 인증 시스템 설계부터 인프라 구축, CI/CD 파이프라인 수립까지 총괄하며 도메인 간의 Bounded Context를 확립하고 배포 생명 주기를 독립적으로 분리하는 미션을 수행했습니다.

**Action**
- **기술 스택 및 아키텍처 현대화**: Kotlin 1.9 및 Spring Boot 3.1을 채택하여 레이어드 아키텍처 기반의 독립 서비스를 구축하고, 기존 세션 방식을 JWT 무상태(Stateless) 인증 체계로 전면 교체했습니다.
- **보안 및 정합성 강화**: RTR(Refresh Token Rotation) 방식을 도입하고 Refresh Token을 Redis와 DB에 이중 저장하는 Cache-Aside 패턴을 적용했으며, 필터 레이어의 예외를 표준 포맷으로 응답하는 커스텀 에러 핸들링을 구현했습니다.
- **성능 및 인프라 최적화**: Aurora MySQL의 읽기/쓰기 분리(Multi-DataSource) 및 ElastiCache Redis 클러스터 모드(3 Master + 3 Replica)를 구축하여 고가용성을 확보하고 고빈도 인증 요청에 따른 부하를 분산했습니다.
- **테스트 및 자동화**: JUnit 5 기반의 Given-When-Then 패턴을 적용해 핵심 비즈니스 로직의 테스트 커버리지를 40% 확보했으며, GitHub Actions와 ArgoCD를 연계하여 테스트 및 배포 전 과정을 자동화했습니다.
- **협업 및 문서화**: springdoc-openapi를 통한 API 명세 자동화 및 Enum 기반의 에러 코드 체계화를 주도했으며, Cursor AI를 활용해 프론트엔드 엔지니어와의 소통 비용을 획기적으로 낮추는 가이드 문서를 구축했습니다.

**Result**
- **시스템 성능 향상**: JWT 기반 무상태 인증 전환을 통해 주문 API 기준 평균 응답 속도를 220ms에서 150ms로 약 30% 이상 개선했으며, 인프라 격리로 독립적인 스케일 아웃이 가능한 구조를 확보했습니다.
- **엔지니어링 생산성 확보**: 물리적 도메인 분리로 빌드 및 컴파일 타임을 단축하고 결합도를 낮추어, 개발자가 인증 로직 수정 시 타 도메인의 영향도 우려 없이 개발에만 몰입할 수 있는 환경을 조성했습니다.
- **운영 안정성 및 신뢰도 구축**: 자동화된 CI/CD 파이프라인과 핵심 로직에 대한 테스트 커버리지 확보를 통해 배포 시 발생할 수 있는 인적 실수를 사전에 차단하고, 일관된 에러 응답 표준을 정립하여 협업 신뢰도를 높였습니다.

### [STUnitas] 문제/풀이 검색 서비스 API 개발

**Situation**  
교육용 커뮤니티 앱에서 타사의 문제·풀이 검색 기능을 벤치마킹해, 유저 반응을 확인한 뒤 본격 개발 여부를 판단할 수 있는 MVP가 필요했습니다. 유사 문제 추천을 위해 검색 엔진이 필요했고, 빠른 구축·배포가 우선인 상황에서 OpenSearch 기반 MSA 형태의 검색 서비스를 별도로 띄워야 했습니다.

**Task**  
백엔드 엔지니어로서 문제·풀이 검색 API를 처음부터 끝까지 단독으로 설계·구현·배포하는 것을 목표로 했습니다. 메인 서비스와 연동되는 RESTful CRUD API를 제공하고, OpenSearch 스코어 알고리즘으로 유사 문제를 찾아주는 검색 기능을 MVP 수준에서 완성하는 데 집중했습니다.

**Action**  
- **서버리스 API 구축**: Flask 기반 RESTful API를 개발하고, Zappa·AWS Lambda·API Gateway로 배포하여 검색 서비스를 MSA 형태로 분리·운영했습니다.
- **OpenSearch 검색 엔진 구성**: OpenSearch(Elasticsearch 호환) 클러스터를 구성하고, 스코어 알고리즘 기반 유사 문제 검색 로직을 구현했습니다.
- **인프라 및 연동**: API Gateway·OpenSearch 등 검색 서비스에 필요한 인프라를 구축하고, 메인 서비스에서 호출할 수 있는 API 형태로 연동했습니다.
- **데이터 마이그레이션 및 장애 대응**: 검색 인덱스 마이그레이션용 스크립트를 작성·적용했으며, 프로덕션 OpenSearch 삭제 사고 발생 시 스냅샷 기반 복구를 수행했습니다.

**Result**  
- **MVP 검색 서비스 배포**: 문제·풀이 검색 API를 단독으로 설계·구현·배포하여, 유저 반응 검증을 위한 MVP 기능을 프로덕션에 반영했습니다.
- **엔드투엔드 구축 경험**: API 설계부터 서버리스 배포·검색 엔진 구성·마이그레이션까지 API 서버 전 과정을 1인으로 수행했습니다.
